# 标标 AI — 系统使用说明书

> 版本：v0.3.0 | 更新日期：2026-03-24

---

## 一、系统概述

**标标 AI** 是一款 AI 驱动的智能标书编制平台，面向工程类（市政道路/房建/水利）、服务类、货物类招标文件的自动化编写。核心能力包括：

- 🤖 **AI 章节生成**：基于招标文件自动生成投标响应内容
- 📚 **知识库管理**：上传历史标书/规范文档，AI 自动检索引用
- 🔄 **变体引擎**：一键生成多份不重复标书，满足围标检测
- 🛡️ **反审标检测**：检测 AI 痕迹，规避评审风险
- 📊 **数据飞轮**：用户反馈驱动 AI 自我进化

## 二、快速上手

### 2.1 登录

1. 访问系统首页（默认 `http://localhost:3001`）
2. 输入邮箱和密码，点击"登录"
3. 开发模式下可点击 **🚀 演示模式 · 一键进入** 跳过登录

### 2.2 工作台（Dashboard）

登录后进入工作台，包含：

| 区域 | 功能 |
|---|---|
| **统计卡片** | 进行中项目数、总项目数、已完成数、AI 反馈量 |
| **快捷入口** | 资料库 / 工艺图谱 / 变体引擎 / 反审标检测 |
| **数据飞轮** | 展示 AI 采纳/修改/拒绝统计及知识回灌情况 |
| **项目列表** | 全部标书项目，可搜索、点击进入编辑 |

## 三、核心功能详解

### 3.1 新建标书项目

1. 点击侧边栏底部 **"新建标书项目"** 或工作台右上角 **"新建标书"**
2. 填写项目名称和类型（工程类/服务类/货物类）
3. 系统自动创建项目并跳转到工作区

### 3.2 工作区（Workspace）

进入项目后进入标书编辑工作区：

- **左侧**：章节目录树，根据招标文件自动生成
- **右侧**：内容编辑区，每个章节可独立生成/编辑
- **AI 生成**：点击章节旁的"生成"按钮，AI 将结合知识库生成内容
- **反馈机制**：对每段 AI 内容可选择 ✅ 采纳 / ✏️ 修改 / ❌ 拒绝

### 3.3 知识库（Library）

路径：侧边栏 → **知识库**

- **上传文档**：支持 PDF/DOCX/TXT 格式，系统自动向量化切片
- **搜索**：输入关键词语义检索相关知识片段
- **统计**：查看总切片数、文件数、结构化表格数

### 3.4 工艺图谱（Craft Library）

路径：侧边栏 → **工艺图谱**

- 内置市政道路 + 房建 248 个工艺节点
- 按工程类型筛选查看标准施工工艺
- AI 生成章节时自动引用匹配的工艺方案

### 3.5 AI 对话

路径：侧边栏 → **AI 对话**

- 输入任何标书编制相关问题
- AI 实时流式回复，支持多轮对话
- 可询问评分标准、施工方案要点、投标策略等

### 3.6 变体引擎（Variants）

路径：侧边栏 → **变体引擎**

- 输入标书原文，设置生成数量
- AI 自动进行同义替换、句式重组、段落重排
- 每份变体的相似度控制在 4%-18% 区间

### 3.7 反审标检测（Anti-Review）

路径：侧边栏 → **反审标**

- 粘贴待检测文本或选择章节
- 系统进行两级检测：
  - **L1 统计检测**：句长分布、词汇丰富度、重复率、连接词密度
  - **L2 基线对比**：与 AI 生成基线的相似度评分
- 返回风险等级（低/中/高/严重）和优化建议

### 3.8 导出

- 在工作区完成编辑后，点击"导出 Word"
- 系统生成格式规范的 .docx 文件自动下载

## 四、系统架构

```
前端 (Next.js 16) ← JWT → 后端 (FastAPI)
                              ↓
                    PostgreSQL + pgvector
                              ↓
                    Gemini 1.5 Flash (LLM)
```

| 层 | 技术栈 |
|---|---|
| 前端 | Next.js 16 + TypeScript + Tailwind v4 |
| 后端 | FastAPI + Python 3.14 |
| 数据库 | PostgreSQL + pgvector 向量检索 |
| AI 引擎 | Gemini 1.5 Flash / Pro |
| 认证 | JWT Token (Bearer) |

## 五、环境配置

### 5.1 后端 `.env`

```bash
DATABASE_URL=postgresql+asyncpg://user:pass@localhost:5434/biaobiao
JWT_SECRET_KEY=your-strong-random-secret-key
CORS_ORIGINS=http://localhost:3000,http://localhost:3001
GOOGLE_API_KEY=your-gemini-api-key
```

### 5.2 启动命令

```bash
# 后端
cd backend && python3 -m uvicorn app.main:app --host 127.0.0.1 --port 8001 --reload

# 前端
cd frontend && npm run dev
```

## 六、常见问题

| 问题 | 解决 |
|---|---|
| 登录后白屏 | 检查 `NEXT_PUBLIC_API_BASE` 是否指向后端 |
| AI 生成无响应 | 确认 `GOOGLE_API_KEY` 配置正确 |
| 知识库上传失败 | 检查文件格式（仅支持 PDF/DOCX/TXT） |
| 导出 Word 报错 | 确认后端已安装 `python-docx` 依赖 |
