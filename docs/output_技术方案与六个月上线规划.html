二、最终实现功能矩阵

2.1 核心七步闭环功能

功能模块 核心能力 技术实现
① 知识库投喂 PDF/Word/TXT 上传 → 自动解析 → 智能切片 → 向量化入库 LangChain Document Loader + pgvector
② 评分点提取 AI 解析招标文件 → 评分矩阵 JSON → 自动映射章节 Gemini 结构化输出 + bid_extract Prompt
③ 大纲生成 评分点驱动目录设计 → 高分项重点展开 → 三级章节结构 bid_outline Prompt + JSON 大纲
④ 七节点流式生成 1轮规划+检索 → 3轮上限Writer-合规-润色-终审迭代 状态机 AsyncGenerator + SSE 推流
⑤ 合规审查 L1 格式规范 + L2 内容覆盖(LLM) + L3 废标预警 ComplianceService 三级引擎
⑥ 多轮自动润色 五级 ≥3轮：术语→文风→逻辑→专业→亮点 PolishService 递进管道
⑦ 反审标检测 L1 统计特征 + L2 AI 基线相似度 → 风险分级 StatisticsAnalyzer + LLM 语义对比
⑧ 变体引擎 同义替换 + 句式重组 → N 套不重复标书（相似度 4~18%） VariantService + 种子随机控制
⑨ Word 导出 自动封面/目录/正文/页眉页脚/中文编号规范排版 python-docx + 样式模板
⑩ AI 对话问答 针对已生成章节的精准问答，SSE 流式回复 RAG + bid_chat Prompt

四、典型应用场景与效率对比

4.1 场景一：常规工程技术标（市政道路/房建）

维度 传统方式 使用本系统 提升
单标书制作工时 3~5 天（1~2 名工程师） 5~8 小时（1 人操作） 减少 85%
可并行处理数量 同期最多 2~3 份 理论无上限 5~10 倍
评分点响应完整性 依赖专家经验，60~75% AI 驱动 ≥90% 提升 20~30%
人力成本（单标） 约 3,000~8,000 元 约 300~500 元（API 成本） 降低 90%

4.2 场景二：服务类标书（物业/安保/保洁）

维度 传统方式 使用本系统 提升
单标书工时 1~2 天 1~3 小时 减少 80%
格式规范正确率 依赖人工检查，出错率 15% 合规引擎自动检查，出错率 ≤3% 减少 80%
围标变体生成 手动改写，每套需 4~6 小时 变体引擎 5 套 ≤1 小时 减少 90%

4.3 场景三：联合发标（多家协作）

  • 多租户架构支持不同工区/项目部独立使用，数据互不可见
  • 统一知识库可由总部集中维护，分公司共享调用
  • 集中管理中标反馈，加速全集团数据飞轮效应

4.4 场景四：SaaS 对外服务（中小建筑商 + 代写机构）

  • 中小建筑商（无标书团队）：订阅后自助生成，降低业务门槛
  • 代写代理机构：大幅提升产能，从”做 10 份”升至”做 100 份”
  • 培训机构：作为教学案例工具，演示 AI 辅助投标流程
