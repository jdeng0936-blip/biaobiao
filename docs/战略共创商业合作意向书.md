## 一、联合愿景：重塑建筑招投标生产力

### 1.1 行业痛点——正在不可逆地恶化

当前国内建筑招投标市场，正同时面临三重绞杀：

| 痛点 | 具体表现 | 对乙方的直接损失 |
|---|---|---|
| **低价内卷白热化** | 同质化竞争者蜂拥而至，利润率持续压缩至 3%~5% | 单纯靠"价格战"已无法建立护城河 |
| **AI 阅标标准严苛** | 评标系统对千篇一律的标书识别率超过 78% | 依赖模板的标书中标率逐年下滑 |
| **高分标书产能瓶颈** | 能写出高分技术标的资深专家稀缺，单人月均仅能完成 2~3 份 | 抢标旺季无法并行作业，坐失商机 |

> **核心判断**：这不是周期性波动，而是行业生产力模式的结构性重塑。下一个五年，标书制作能力将从"专家手艺"升维为"可规模化复制的数字武器"。

### 1.2 共创愿景——颠覆级的能力组合

```
乙方十年行业 Know-how          甲方前沿 AI 引擎
  ├── 真实中标标书数据库    +    ├── 七节点智能生成管线
  ├── 资深造价师专家判断    +    ├── 三级合规审查引擎
  └── 行业客户与渠道网络    +    └── 反AI阅标检测技术
              ↓                            ↓
         ═══════════════════════════════════
         行业颠覆级武器：智能标书制造引擎
         — 中标率提升目标：+25%~40%
         — 单标书制作工时：从 3~5 天 → 5 小时
         — 可并行处理标书数量：从 2~3 份 → 无上限
```

双方以"**联合发起人**"身份共同缔造这一产品，乙方将从"优秀的标书制作者"跃升为"向全行业输出能力的数字化赋能方"。

## 四、商业化阶段：内部赋能与对外 SaaS 分润（Phase 2+）

### 4.1 内部提效——乙方获得的第一层价值（数字资产化）

产品正式上线后，乙方获得：

- **企业版永久使用权**：无需支付任何订阅费用，乙方内部员工不限席位使用
- **专属数据空间隔离**：乙方数据独立存储，其他租户不可访问
- **优先迭代权**：乙方业务场景需求享有产品迭代最高优先级
- **预期降本效果**：
  - 每份标书制作工时：从 **3~5 天** → **5 小时**（减少 85%）
  - 可并行处理标书数量：从 **2~3 份/月** → **理论无上限**
  - 高级标书专家工时解放：从重复撰稿 → 聚焦策略性决策

> **换算成人效价值**：如乙方每年参与投标 50 个项目，节省工时折算人力成本约 **60~90 万元/年**（3 名资深工程师全年薪酬对标）。

### 4.2 对外商业化——乙方获得的第二层价值（持续分润）

当平台作为标准 SaaS 产品推向全国中小型建筑商与代写代理机构时，乙方以**"行业数据与知识共创联合发起人"**身份，享有以下商业权益：

| 商业模式 | 分润方式 | 说明 |
|---|---|---|
| **SaaS 订阅收入** | 对外销售净利润的 **25%** | 按季度结算，数据透明可审计 |
| **区域独家代理权** | **本省**区域优先代理权，代理佣金比例 **15%** | 乙方可在省内发展下游经销商网络 |
| **行业白标授权** | 向同行建筑商提供私有化部署，乙方收取渠道费 | 利用乙方行业信任背书快速变现 |

**市场空间测算（保守估计）**：

```

年营收规模：1,200 × 1.8 万 = 2,160 万元/年
```

乙方由此完成从"**传统施工承包方**"到"**行业数字化赋能平台联合发起人**"的战略跨越。

## 五、核心保障：军工级数据隔离与知识产权归属

### 5.1 数据安全——真实标书数据永不离境

这是双方合作最核心的信任基石。甲方在技术架构层面，设计并实现了**"本地统一脱敏网关"**机制，以工程手段而非合同承诺来保障乙方数据安全：

```
乙方本地机房（物理安全边界）
├── 原始文件存储（加密静态存储）
│    └── 真实标书、评分标准、历史数据
├── 本地脱敏网关（NER 命名实体识别）
│    └── 公司名、项目地址、报价金额 → 脱敏为占位符
│         "某建工集团" / "某市政道路项目" / "¥X,XXX 万"
├── 本地向量数据库（pgvector）
│    └── 语义切片后的知识条目永久存储于本地
└── 仅脱敏后的文本片段经加密信道上传云端 LLM
         ↓ 云端生成内容返回
本地网关逆映射还原 → 真实内容重新插入 → 交付给用户界面

====== 技术承诺 ======
原始数据在任何情况下，均不离开乙方机房物理边界。
甲方研发团队无法访问乙方原始标书内容。
```

**可验证性**：甲方承诺提供完整源代码脱敏网关模块供乙方技术团队审计，或委托第三方安全机构进行代码审计，确保技术承诺的可核查性。

### 5.2 知识产权归属——明确无争议的产权边界

| 资产类型 | 归属方 | 备注 |
|---|---|---|
| **系统底层代码** | 甲方（100%）| 含 AI 生成引擎、合规审查引擎、润色管道、前端全栈 |
| **系统软件著作权** | 甲方（100%）| 甲方单独申请并持有 |
| **乙方原始历史标书** | 乙方（100%）| 永久归属，随时可要求销毁本地数据库中的副本 |
| **基于乙方数据训练的 AI 模型权重** | 双方共有（各 50%）| 独立申请，任何一方不得单独转让 |
| **联合品牌与联合发起人资格** | 双方共有 | 对外宣传须双方授权 |
| **产品对外销售的商标** | 甲方主导申请，乙方享有特许使用权 | — |
