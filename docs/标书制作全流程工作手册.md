## Step 1：投喂资料（知识库构建）

> **目的**：教会 AI 你的专业知识、历史经验和行业规范

### 1.1 该投喂什么资料？

| 资料类型 | 举例 | 作用 |
|---|---|---|
| **中标范文** ✨ | 过去中标的技术标书 PDF/Word | AI 学习你的写作风格、技术方案结构 |
| **招标文件** | 本次招标文件 | 提取评分标准和技术要求 |
| **施工规范** | 《公路工程施工质量验收规范》等 | AI 引用正确的规范条款编号 |
| **企业资料** | 公司简介、业绩清单、资质证书 | 自动填写企业实力章节 |
| **施工组织设计** | 历史项目的施组方案 | 复用成熟的技术方案框架 |
| **工艺标准** | 混凝土浇筑工艺、钢筋绑扎流程 | 生成专业的施工工艺描述 |

### 1.2 操作步骤

```
侧边栏 → 知识库 → 点击"上传文档"
                  ↓
         选择 PDF / Word / TXT 文件
                  ↓
         系统自动完成：
         ① 文档解析（提取正文内容）
         ② 智能切片（按段落/章节拆分）
         ③ 向量化入库（语义索引）
                  ↓
         上传完成，显示"已生成 N 个知识切片"
```

### 1.3 高质量知识投喂技巧

> [!TIP]
> **质量 > 数量**。10 份高质量中标范文 > 100 份垃圾文档

1. **优先投中标范文**：AI 会优先学习评分最高的内容风格
2. **按项目类型分类**：市政道路、房建、水利分批上传，AI 会按类型匹配
3. **保留格式结构**：Word 比 PDF 解析效果更好（保留标题层级）
4. **删除无用页**：去掉目录、封面、空白页，减少噪音
5. **投喂行业规范**：GB/T、JTG 等规范文档让 AI 引用有据

## Step 2：提取评分点（评分解析）

> **目的**：从招标文件中提取评审打分项，确保标书精准响应

### 2.1 操作步骤

```
新建项目 → 进入工作区
              ↓
    上传/粘贴招标文件评分标准部分
              ↓
    系统自动提取：
    ① 评分大类（技术方案、施工组织、质量保证...）
    ② 各项分值权重
    ③ 对应评审标准
              ↓
    生成评分点矩阵 → 一一对应到标书章节
```

### 2.2 AI 如何利用评分点

评分点直接驱动章节生成——**有分值的地方就多写，没分值的地方精简**：

```
评分项：施工组织设计（25分）
  ├── 总体施工部署（8分）→ AI 重点展开，2000+ 字
  ├── 进度计划（7分）→ AI 生成甘特图描述 + 关键里程碑
  ├── 资源配置（5分）→ AI 生成人员/机械配置表
  └── 冬雨季施工（5分）→ AI 生成专项方案
```

## Step 3：AI 生成标书（章节编写）

> **目的**：逐章生成高质量技术标书内容

### 3.1 操作步骤

```
工作区 → 选择任意章节 → 点击"AI 生成"
                           ↓
          AI 自动执行（后台全自动）：
          ① 检索知识库中匹配的范文片段
          ② 匹配评分点中的分值权重
          ③ 查找工艺图谱中对应施工工艺
          ④ 结合以上信息，流式生成章节内容
                           ↓
          内容逐字显示在编辑区
                           ↓
          用户审阅后操作：
          ✅ 采纳 → 直接保存
          ✏️ 修改 → 编辑后保存（AI 学习你的修改偏好）
          ❌ 拒绝 → 重新生成
```

### 3.2 生成高质量内容的关键

| 策略 | 做法 | 效果 |
|---|---|---|
| **知识库越充实，生成越精准** | 投喂 5+ 份同类型中标范文 | AI 输出风格贴近中标水平 |
| **评分点越明确，响应越到位** | 上传完整评分标准 | 每段内容精准对应评分项 |
| **反馈越积极，AI 越聪明** | 及时采纳/修改/拒绝 | 飞轮持续优化生成策略 |
| **分章节生成，逐个打磨** | 不要一次全部生成 | 可针对性调整每章风格 |

## Step 4：合规审查（格式+内容+废标预警）

> **目的**：确保标书格式规范、内容合规、不触发废标条件

### 4.1 三级审查机制

```
已内嵌在七节点管线中（Node 4），生成后自动执行
也可通过 API 独立调用：/api/v1/compliance/check

  L1 格式规范（纯规则引擎，毫秒级）：
    · 编号连续性（一、二、三...不跳号不重号）
    · 章节字数统计（是否达到评分最低要求）
    · 规范引用格式校验（GB/T xxxx-xxxx 年份合理性）
    · 口语化表达检测（"大概""差不多"等 17 个黑名单词）
    · 过短段落预警

  L2 内容合规（LLM 语义审查，秒级）：
    · 评分点响应性矩阵（是否每个评分点都有实质覆盖）
    · 关键技术要求覆盖度
    · 数据一致性（工期/金额/人数全文统一）

  L3 废标风险预警（硬规则）：
    · 投标人名称一致性（"XX公司"占位符残留）
    · 禁止条款检测（"不承担任何责任"等）
    · 工期/造价超出招标范围
                      ↓
  返回合规分数（0-100）+ 问题列表 + 修复建议
  致命问题 → 自动触发退档重写
```

## Step 5：多轮润色（三轮递进式精修）

> **目的**：自动化多轮迭代打磨，确保专业性、一致性和竞争力

### 5.1 五级润色管道

```
Round 1: L1 术语规范化 + L2 文风一致性 — 基础规范化
Round 2: L3 逻辑连贯性 + L4 专业深化   — 内容增强
Round 3: L5 亮点提炼                   — 竞争力提升
```

| 级别 | 做什么 | 举例 |
|---|---|---|
| L1 术语 | 统一专业用词 | "水泥沙浆" → "水泥砂浆" |
| L2 文风 | 第三人称/陈述语气/书面化 | "我们" → "本公司" |
| L3 逻辑 | 补充过渡句/因果论述 | 加入"基于上述分析..." |
| L4 专业 | 增加规范引用/技术参数 | 加入 GB50300-2013 条款 |
| L5 亮点 | 针对高分评分点强化差异化 | "采用BIM技术实现全过程管控" |

### 5.2 润色效果

```
初稿（AI 生成原文）
    ↓ Round 1: 术语统一 + 文风规范化
    ↓ Round 2: 逻辑增强 + 专业深化
    ↓ Round 3: 亮点提炼（对应评分点）
终稿 — 质量分 85+ / 100
```

## Step 6：反审标检测（风险消除）

> **目的**：确保标书无法被检测出 AI 痕迹

### 4.1 操作步骤

```
侧边栏 → 反审标 → 粘贴/选择章节内容
                        ↓
         系统两级检测：
         L1 统计检测：
           · 句长分布（AI 倾向句长均匀）
           · 词汇丰富度（AI 用词偏模板化）
           · 重复率（AI 段落间重复度高）
           · 连接词密度（AI 爱用"首先、其次、最后"）
         
         L2 基线对比：
           · 与 AI 典型输出的相似度评分
                        ↓
         返回风险等级 + 优化建议：
         🟢 低风险 → 直接使用
         🟡 中风险 → 部分修改
         🔴 高风险 → 需要重写或人工润色
         🚨 严重   → 变体引擎重新生成
```

### 4.2 降低风险的技巧

1. **先生成，后检测**：所有章节生成完毕后统一检测
2. **高风险章节用变体引擎重做**：自动同义替换降低相似度
3. **人工加入"瑕疵"**：适当加入口语化表达、个人观点，破坏 AI 规律性
4. **引用真实数据**：插入项目实际数据（工期、造价、人数），AI 不会编造这些

## Step 5：变体输出（多套标书）

> **目的**：一份原稿 → N 份不重复标书（满足围标检测要求）

### 5.1 操作步骤

```
侧边栏 → 变体引擎
              ↓
    粘贴标书原文 → 设置生成数量（如 5 份）
              ↓
    AI 自动执行：
    ① 同义词替换（建议→推荐，采用→使用）
    ② 句式重组（主动↔被动，长句↔短句）
    ③ 段落重排（调整论述顺序）
    ④ 数据微调（在合理范围内调整技术参数表述）
              ↓
    输出 N 份变体，每份相似度 4%~18%
              ↓
    每份变体 → 反审标检测 → 确认安全后使用
```

### 5.2 多套标书的质量保障

```
原稿（100分品质）
  ├── 变体 A：97分（语序微调，含义不变）
  ├── 变体 B：95分（同义替换，专业性不降）
  ├── 变体 C：96分（段落重排，逻辑清晰）
  └── 变体 D：94分（句式重组，风格略异）
```

```
用户反馈
  ↓

  ↓

```
